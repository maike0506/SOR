CONFIGURACION DEL SERVIDOR SAN Y LOS CLIENTES SAN


instalamos el paquete al servidor san


sudo apt update
sudo apt install tgt

creamos los target en el siguiente fichero

sudo nano /etc/tgt/targets.conf

modificamos las siguientes lineas en ese fichero " anadiremos un iqn ( iscsi)"


default-driver iscsi

< target iqn.michael com remotedisk office:target1>

backing-store /dev/sda


initiator-addresses 172.16.0.4

< / target>













  1. explicacion de cada linea que modificamos en ese fichero

  • backing-store ; disdo donde vamos a almacena

  • initiato-address : direccion ip de la maquina que usara nuestro volumen

SIGUIENTE PASO REINICIAR EL SERVIDOR

  1. con el siguiente comando puedes parar el servicio de tgt

    service tgt stop


  2. con el siguiente comando puedes reiniciar el servicio de tgt

    service tgt start


  3. con el siguiente comando puedes ver el estado del servicio tgt

    service tgt status




CONFIGURACION EN LA MAQUINA CLIENTE EN NUESTRO CASO VA HACER UNA MAQUINA WINDOWS


  1. panel de contro > iscsi > configurar inicados iscsi

  • pulsamos en "SI" en la ventana que nos aparece

  1. Enste paso nos aparece la ventana de propiedades >


  • escribimos la ip del servidor SAN > configuracion rapida

Nos dirijimos al fichero de administracion de discos

  • aqui le tenemos que asignar una letra en nuestro caso le asignamos la Z >> siguiete

  • aqui le tenemos que asignar un nombre >> MICAHEL >> siguiente
