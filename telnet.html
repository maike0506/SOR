COMO CONFIGURAR TELNET

PRIMERA PASO INSTALAR EL SIGUIENTE PAQUETE QUE ES EL DEMONIO DE TELENET


$ sudo apt update

$ sudo apt install openbsd-inetd

cuando intalemos este paque tenemos que mirar su fichero de configuracion para ver si tiene habilitado el telnet y no lo tiene comentado

$ sudo nano /etc/inetd.conf


en este fichero lo que tenemos que mirar es que la linea del telnet no este comentada basicamente
si esta comentado le quitamos la # y luego tendremos que reiniciar el demonio de openbsd-inetd con el siguiente comando


sudo /etc/init.d/openbds-inetd restart



instalacion del demonio telnet


$ sudo apt install telnetd

PARA EJECUTAR EL SIGUIENTE COMANDO TENDREMOS QUE VERIFICAR QUE TENGAMOS INTALADO EL "net-tools"



which net-tools

con este comando verificaremos que tengamos instalado el net-tools



sudo apt update
sudo apt install net-tools

ahora si podremos ejectura el comando de netstat

$ netstat -ltpn
$ netstat -ltp


  • netstat -ltp : esto solo nos mostrar el servicio que tenemos abierto no nos mostrara el numero de puerto de nuestro servicio como por ejemplo el de telnet que se abre por el puerto 23


  • netstat -ltpn: este comando si nos mostrar el numero de puerto pero si el nombre del servicio pero ya sabremos que si es el 23 es el de telnet


COMANDOS PARA CAMBIAR EL PUERTO DEL TELNET



sudo nano /etc/services

una vez cambiado el puerto tendremos que reiniciar el demonio de openbsd-inetd con el siguiente comando



sudo /etc/init.d/openbsd-inetd restart

concretado la instalacion y que el puerto este abierto probamos a conectarno con la siguiente sintaxis

telnet usuario@(ip de tu servidor )(puerto)

para conectarnos de un usuario origen a un usuario destino lo hacemos de la siguiete manera



$michael@michael$ telnet -l michael1 michael2

aqui lo que estamo haciendo es conectarno de michael1 a michael2 atraves de michael que es mi usuario principal

INSTALACION DE FTP


sudo apt update
sudo apt install vsftpd

para entrar al fichero de configuracion ftp tenemos que ejecutar el siguiente comando



sudo nano /etc/vsftpd.conf

para reiniciar el servicio de ftp y ver el estado tamien tienes los siguientes comandos


sudo service vsftpd restart

sudo service vsftpd status

CONFIGURACION EN EL CLIENTE FTP

en el cliente no tendremos que instalar nada por que ya viene con los sistemas linus solo tendremos que conectarnos al servidor ftp con la siguiente sintaxis



ftp < ip servidor>

cuando estemos realizando la conexion nos pedira que nos logeomos con el nombre de nuestro usuario cliente y nuestra contrasena como usuarios clientes



PARA VER POR QUE PUERTO ESTA ESCUCHANDO EL FTP

netstat -ltpn

vemos que el ftp se escucha por el PUERTO 21

para cerrar la conecion con dicho usuario utilizamos los siguiente



quit
bye


este comando nos permite cambiar de directorio en nuestro host local es decir desde nuestro cliente para subir el archivo al servidor ftp



lcd < nombre de nuestro directorio local>

para descargar un archivo utilizamos el siguiete comando



get < nombre del archiv>

para descargar varios archivos utilizamos el siguietne comando



mget < nombre de archivo de los archivos >

para subir un archivo a un cliente a un servidor utiilizamos el siguiente comando


put < nombre del archivo>

para subir varios archivos del cliente al servidor utilizamos el siguiente comando



mput < nombre de losarchivo>
